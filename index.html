stylecow

Node package to parse and manipulate your CSS code.
Based on the original PHP library

Project maintained by oscarotero | MIT license

Why another CSS preprocessor?

The main purpose of Stylecow is to bring more CSS support to all browsers. You write CSS and you get CSS. You don't have to learn another different language such LESS, SASS, etc. Stylecow converts your code to be more compatible with all browsers throught plugins without writing any non-standard code. So you can use Stylecow just to fix your modern CSS code and make it compatible with old browsers. And if you stop using Stylecow, your CSS code will remain CSS code.

How to use?

First, you need to install the library using npm. Use -g to install as global:

$ npm install stylecow -g

Now you can convert any css file:

$ stylecow styles.css

And save the result in another file:

$ stylecow styles.css > compatible.styles.css

Execute stylecow without arguments to view all plugins and options availables

$ stylecow

Options

By default, stylecow brings support for the following minimal version browsers:

  • Explorer 8.0
  • Firefox 15.0
  • Chrome 25.0
  • Safari 5.0
  • Opera 12.0
  • Android 4.0
  • iOs 5.0

But you can change the minimal browser support:

$ stylecow styles.css --firefox=3.0 --explorer=6.0

Some plugins are disabled by default because they are not "safe" plugins and can change some non desired aspects of the design. For example, ieLinearGradient plugin emulates the background gradient in explorer using the filter property, but sometimes it's better to use a plain colors because the abuse of filters in explorer affects to render performance. You can enable/disable plugins with the --enable and --disable options:

$ stylecow styles.css --enable=ieLinearGradient --disable=ieTransform,ieBackgroundAlpha

Plugins

The complete list of plugins:

color

Enabled by default

Support for color() function. Manipulate color dinamically. Changes some values in absolute or relative way. This function may change in a future due to the possible implementation a color() function in CSS4 color module.

color(<color> [, alpha|tint|hue|saturation|light|red|green|blue])
Mix various colors:
color(yellow blue #222 rgba(32, 34, 67, 0.5));

Manipulate the color using more arguments (tint, alpha, red, blue, green, light, saturation, hue):
color(yellow blue, tint:45, alpha:0.5, blue:+10, light: 25)

Shorcuts for tint and alpha:
color(rgb(34, 56, 78), 50);
color(rgb(34, 56, 78), 0.5);

Example:
.foo {
    background: color(rgb(34, 56, 78), tint:50);
}

ffBorderRadius

Enabled by default and applied for firefox<13

Add the old border-[*]-radius syntax for firefox.

You write:

.foo {
    border-top-left-radius: 10px;
}

And stylecow converts to:

.foo {
    -moz-border-radius-topleft: 10px;
    border-top-left-radius: 10px;
}

ieBackgroundAlpha

Disabled by default and applied for explorer<9

Add support of rgba/hsla colors in backgrounds for explorer using ms filters more info

You write:

.foo {
    background: rgba(0, 0, 0, 0.5);
}

And stylecow converts to:

.foo {
    background: rgba(0, 0, 0, 0.5);
    filter: progid:DXImageTransform.Microsoft.gradient(startColorStr='#80000000', endColorStr='#80000000');
}

ieClip

Enabled by default and applied for explorer<8

Fix the old syntax of clip for ie 6 and 7

You write:

.foo {
    clip: rect(5px, 40px, 45px, 5px);
}

And stylecow converts to:

.foo {
    clip: rect(5px, 40px, 45px, 5px);
    *clip: rect(5px 40px 45px 5px);
}

ieFloat

Enabled by default and applied for explorer<7

Fix the double margin bug in ie6 on float block elements

You write:

.foo {
    float: left;
}

And stylecow converts to:

.foo {
    float: left;
    _display: inline;
}

ieInlineBlock

Enabled by default and applied for explorer<8

Add support for display:inline-block in explorer 6 and 7

You write:

.foo {
    display: inline-block;
}

And stylecow converts to:

.foo {
    display: inline-block;
    *zoom: 1;
    *display: inline;
}

ieLinearGradient

Disabled by default and applied for explorer<10

Emulate the linear-gradient background in explorer using ms filters

You write:

.foo {
    background: linear-gradient(top, red, blue);
}

And stylecow converts to:

.foo {
    background: linear-gradient(top, red, blue);
    filter: progid:DXImageTransform.Microsoft.gradient(startColorStr='#FF0000', endColorStr='#0000FF');
}

ieMinHeight

Enabled by default and applied for explorer<7

Add support for min-height property to ie 6, more info

You write:

.foo {
    min-height: 200px;
}

And stylecow converts to:

.foo {
    min-height: 200px;
    _height: 200px;
}

ieOpacity

Enabled by default and applied for explorer<9

Add opacity support for explorer using filters

You write:

.foo {
    opacity: 0.5;
}

And stylecow converts to:

.foo {
    opacity: 0.5;
    filter: alpha(opacity=50);
}

ieTransform

Enabled by default and applied for explorer<9

Add ms filters to emulate some 2d css transformations: rotate(), scale(-1, -1), scalex(-1) and scaley(-1)

You write:

.foo {
    transform: rotate(45deg);
}

And stylecow converts to:

.foo {
    transform: rotate(45deg);
    filter: progid:DXImageTransform.Microsoft.Matrix(sizingMethod="auto expand", M11 = 0.70710678118655, M12 = -0.70710678118655, M21 = 0.70710678118655, M22 = 0.70710678118655);
}

initial

Enabled by default and applied for firefox<19, chrome<1, safari<1.2 and opera<15

Add support for the "initial" value

You write:

.foo {
    background-position: initial;
    height: initial;
}

And stylecow converts to:

.foo {
    background-position: 0 0;
    height: auto;
}

linearGradient

Enabled by default and applied for explorer<10, chrome<26, safari<6.1, opera<12.1, ios<7.0 and android<4.4

Fallback for browsers supporting the olders syntax of linear-gradient and -webkit-gradient

You write:

.foo {
    background: linear-gradient(to bottom, red, blue);
}

And stylecow converts to:

.foo {
    background: -webkit-gradient(linear, left top, left bottom, from(red), to(blue));
    background: -o-linear-gradient(top, red, blue);
    background: -webkit-linear-gradient(top, red, blue);
    background: -moz-linear-gradient(top, red, blue);
    background: linear-gradient(to bottom, red, blue);
}

matches

Enabled by default

Add support for CSS4 :matches() selector in all browsers, more info

You write:

.foo :matches(h1, h2, h3, h4, h5, h6) a {
    color: blue;
}
.foo :matches(article, section) header :matches(h1, h2) {
    color: black;
}

And stylecow converts to:

.foo h1 a, .foo h2 a, .foo h3 a, .foo h4 a, .foo h5 a, .foo h6 a {
    color: blue;
}
.foo article header h1, .foo article header h2, .foo section header h1, .foo section header h2 {
    color: black;
}

nestedRules

Enabled by default

Add support for nested rules, more info

You write:

.foo {
    color: blue;

    a {
        color: red;

        &:hover {
            color: green;
        }
    }
}

And stylecow converts to:

.foo {
    color: blue;
}
.foo a {
    color: red;
}
.foo a:hover {
    color: green;
}

rem

Enabled by default and applied for firefox<3.6, explorer<9.0, safari<5.0, opera<11.6 and ios<4.0

Add a fallback in pixels for browsers not supporting rem values

You write:

.foo {
    font-size: 2rem;
}

And stylecow converts to:

.foo {
    font-size: 32px;
    font-size: 2rem;
}

You can change the default value of 1rem (16px) defining a new font-size in :root or html. For example, you write:

html {
    font-size: 1.2em;
}
.foo {
    font-size: 1rem;
}

And stylecow converts to:

html {
    font-size: 1.2em;
}
.foo {
    font-size: 19.2px;
    font-size: 1rem;
}

variables

Enabled by default

Add variables using standard var() function, more info. For global variables (available in all properties), you have to define them in the selectors :root or html. Use nested rules to create scoped variables.

var(varname [, default])

You write:

:root {
    var-mycolor: red;
}

.foo {
    color: var(mycolor);
}

.foo h2 {
    var-mycolor: blue;

    color: var(mycolor);
    font-size: var(mySize, 24px);
}

And stylecow converts to:

.foo {
    color: red;
}

.foo h2 {
    color: blue;
    font-size: 24px;
}

vendorPrefixes

Enabled by default

Add automatically the vendor prefixes for common browsers. Uses the support configuration to detect whether the vendor prefix is needed or not in each case.

You write:

.foo {
    box-sizing: border-box;
    user-select: none;
}

.foo ::selection {
    background: red;
}

And stylecow converts to:

.foo {
    -moz-box-sizing: border-box;
    box-sizing: border-box;
    -webkit-user-select: none;
    -moz-user-select: none;
    -ms-user-select: none;
    user-select: none;
}

.foo ::selection {
    background: red;
}

.foo ::-moz-selection {
    background: red;
}
