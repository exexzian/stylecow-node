stylecow

Node package to parse and manipulate your CSS code.
Based on the original PHP library

Project maintained by oscarotero | MIT license

Why another CSS preprocessor?

The main purpose of Stylecow is to bring more CSS support to all browsers. Stylecow converts your code to be more compatible with all browsers adding vendor prefixes, fixing browser specific bugs, emulating some non-supporting css properties, etc.

You can also play with new (or future) css features such variables, nested rules or color manipulation using the w3c syntax. If you need more power, you can use other more powerful tools like SASS or LESS and use Stylecow only to fix your code for more compatibility.

How to use?

First, you need to install the library using npm. Use -g to install it globaly:

$ npm install stylecow -g

Now you can convert any css file:

$ stylecow styles.css

And save the result in another file:

$ stylecow styles.css > compatible.styles.css

Execute stylecow without arguments to view all plugins and options availables

$ stylecow

Programatic usage:

var stylecow = require('stylecow');
var css = new stylecow();

css.loadFile('my-styles.css');
css.transform();
css.output('dist/my-styles.css');

Options

By default, stylecow brings support for the following minimal version browsers:

  • Explorer 8.0
  • Firefox 15.0
  • Chrome 25.0
  • Safari 5.0
  • Opera 12.0
  • Android 4.0
  • iOs 5.0

But you can change the minimal browser support:

$ stylecow styles.css --firefox=3.0 --explorer=6.0

Some plugins are disabled by default because they are not "safe" plugins and can change some non desired aspects of the design. For example, ieLinearGradient plugin emulates the background gradient in explorer using the filter property, but the abuse of filters in explorer can affect to performance and have conflicts with other properties such border-radius. You can enable/disable plugins with the --enable and --disable options:

$ stylecow styles.css --enable=ieLinearGradient --disable=ieTransform,ieBackgroundAlpha

Some plugins have specific settings. Use the syntax --pluginName.settingName=value:

$ stylecow styles.css --enable=ieLinearGradient --disable=ieTransform,ieBackgroundAlpha --removeComments.important=true

You can use css comments to enable/disable/configure plugins. This is useful to change the configuration for specific parts of your css. Just create css comments starting by "stylecow":

/* stylecow --enable=ieLinearGradient --firefox=20.0 */

.foo {
    background: linear-gradient(to bottom, black, blue);
}

/* stylecow --disable=ieLinearGradient */

The --output option saves the result in a css file.

$ stylecow styles.css --output=compatible.styles.css

Or you can define an output file using a second arguments:

$ stylecow styles.css compatible.styles.css

The --watch option executes the transformation when the main file (or any of the imported files) changes and save the result in the output file. This allows combine stylecow with browser-sync or any other similar tool.

$ stylecow styles.css compatible.styles.css --watch

Load options from json

Other more portable way to set the options is using a json file. Just create a file with the same name than the css appending the ".json" suffix. For example to set the options for styles.css, you must create styles.css.json in the same directory.

An example of configuration file can be the following:

{
    "explorer": 10.0,
    "firefox": 20.0,
    "chrome": 30.0,
    "safari": 7.0,
    "opera": 16.0,
    "android": 4.0,
    "ios": 6.0,
    "enable": ["ieLinearGradient", "ieTransform"]
}

Any configuration value provided in command line will overwrite the json value (if exists).

In the config file you can also define the output code style. By default is "default", but "minify" is also available:

{
    "code": "minify"
}

Or a more fine tunning:

{
    "code": {
        "rulesetIndent": "\t",
        "selectorJoiner": ", ",
        "ruleColon": ": ",
        "ruleStart": "\t",
        "ruleEnd": ";\n",
        "rulesetStart": " {\n",
        "rulesetEnd": "}\n"
    }
}
  • rulesetIndent: The string used for the indentation of the nested rulesets. By default it's a tab but you can provide spaces (or any string you want).
  • selectorJoiner: The string used to join various selectors. By default is a comma following a space (span, strong, em).
  • ruleColon: The string used to join the property name and value in each rule. By default is a colon following a space (color: red)
  • ruleStart: The string used at begining of each rule. By default it's a tab (to indent all rules in the ruleset)
  • ruleEnd: The string used in the end of each rule. By defaul is a semicolon following a line break (to write each rule in a diferent line)
  • rulesetStart: The string used at begining of each ruleset (just after the selector). By default it's a space following the open bracket and a new line.
  • rulesetEnd: The string used in the end of each ruleset. By default is the closed bracket following a line break.

Plugins to play with new css features

  • color Enabled by default

    Support for color() function. Manipulate color dinamically. Changes some values in absolute or relative way. This function may change in a future due to the possible implementation a color() function in CSS4 color module.

    color(<color> [, alpha|tint|hue|saturation|light|red|green|blue])
    Mix various colors:
color(yellow blue #222 rgba(32, 34, 67, 0.5));

Manipulate the color using more arguments (tint, alpha, red, blue, green, light, saturation, hue):
color(yellow blue, tint:45, alpha:0.5, blue:+10, light: 25)

Shorcuts for tint and alpha:
color(rgb(34, 56, 78), 50);
color(rgb(34, 56, 78), 0.5);

Example:
.foo {
    background: color(rgb(34, 56, 78), tint:50);
}

  • matches Enabled by default

    Add support for CSS4 :matches() selector in all browsers, more info

    You write:

    .foo :matches(h1, h2, h3, h4, h5, h6) a {
    color: blue;
}
.foo :matches(article, section) header :matches(h1, h2) {
    color: black;
}

    And stylecow converts to:

    .foo h1 a, .foo h2 a, .foo h3 a, .foo h4 a, .foo h5 a, .foo h6 a {
    color: blue;
}
.foo article header h1, .foo article header h2, .foo section header h1, .foo section header h2 {
    color: black;
}

  • nestedRules Enabled by default

    Add support for nested rules, more info

    You write:

    .foo {
    color: blue;

    a {
        color: red;

        &:hover {
            color: green;
        }
    }
}

    And stylecow converts to:

    .foo {
    color: blue;
}
.foo a {
    color: red;
}
.foo a:hover {
    color: green;
}

  • variables Enabled by default

    Add variables using standard var() function, more info. For global variables (available in all properties), you have to define them in the selectors :root or html. Use nested rules to create scoped variables.

    var(varname [, default])

    You write:

    :root {
    --mycolor: red;
}

.foo {
    color: var(--mycolor);
}

.foo h2 {
    --mycolor: blue;

    color: var(--mycolor);
    font-size: var(--mySize, 24px);
}

    And stylecow converts to:

    .foo {
    color: red;
}

.foo h2 {
    color: blue;
    font-size: 24px;
}

Plugins with useful utilities

  • import Enabled by default

    Includes the @import files with relative path (not absolute or external)

    You write:

    @import "my-styles.css";
.foo {
    color: blue;
}

    And stylecow converts to:

    .imported-foo {
    font-size: 2em;
}
.other-imported-foo {
    background: blue;
}
.foo {
    color: blue;
}

  • removeComments Enabled by default

    Removes all CSS comments. By default only removes non-important comments (All comments starting by "!" are importants). Use the setting removeComments.important=true to remove also the important comments.

    You write:

    .foo {
    color: blue; /* My favourite color */
}
/* But green is not bad too */
.foo:hover {
    color: green;
}

    And stylecow converts to:

    .foo {
    color: blue;
}
.foo:hover {
    color: green;
}

  • sourceMap Disabled by default

    Adds a comment before each ruleset element with the original filename and column number.

    You write:

    .foo {
    user-select: none;
}
.foo:hover {
    color: green;
}

    And stylecow converts to:

     /* styles.css:1 */
.foo {
    -webkit-user-select: none;
    -moz-user-select: none;
    -ms-user-select: none;
    user-select: none;
}

/* styles.css:4 */
.foo:hover {
    color: green;
}

Plugins to fix/normalize css implementations

  • vendorPrefixes Enabled by default

    Add automatically the vendor prefixes for common browsers. Uses the support configuration to detect whether the vendor prefix is needed or not in each case.

    You write:

    .foo {
    box-sizing: border-box;
    user-select: none;
}

.foo ::selection {
    background: red;
}

    And stylecow converts to:

    .foo {
    -moz-box-sizing: border-box;
    box-sizing: border-box;
    -webkit-user-select: none;
    -moz-user-select: none;
    -ms-user-select: none;
    user-select: none;
}

.foo ::selection {
    background: red;
}

.foo ::-moz-selection {
    background: red;
}

  • linearGradient Enabled by default

    explorer < 10.0 chrome < 26.0 safari < 6.1 opera < 12.1 ios < 7.0 android < 4.4

    Fallback for browsers supporting the olders syntax of linear-gradient and -webkit-gradient

    You write:

    .foo {
    background: linear-gradient(to bottom, red, blue);
}

    And stylecow converts to:

    .foo {
    background: -webkit-gradient(linear, left top, left bottom, from(red), to(blue));
    background: -o-linear-gradient(top, red, blue);
    background: -webkit-linear-gradient(top, red, blue);
    background: -moz-linear-gradient(top, red, blue);
    background: linear-gradient(to bottom, red, blue);
}

  • calc Enabled by default

    Normalizes the calc() function that requires spaces around binary "+" and "-" operators. Internet explorer (in accordance with the spec) does not accept calculations without spaces. More info

    You write:

    .foo {
    width: calc(100%-30px);
}

    And stylecow converts to:

    .foo {
    width: calc(100% - 30px);
}

  • ieFlex Enabled by default

    explorer < 11.0

    Add explorer 10 old syntax for css flexbox.

    You write:

    .foo {
    display: flex;
    justify-content: flex-start;
}

    And stylecow converts to:

    .foo {
    display: -ms-flexbox;
    display: flex;
    -ms-flex-pack: start;
    justify-content: flex-start;
}

  • ffBorderRadius Enabled by default

    firefox < 13.0

    Add the old border-[*]-radius syntax for firefox.

    You write:

    .foo {
    border-top-left-radius: 10px;
}

    And stylecow converts to:

    .foo {
    -moz-border-radius-topleft: 10px;
    border-top-left-radius: 10px;
}

  • pseudoElements Enabled by default

    explorer < 9.0

    Ie 8 do not support ::before, ::after, ::first-line and ::first-letter pseudoelements with two colons but support :before, :after, :first-line and :first-letter (one colon).

    You write:

    .foo::before {
    content: "bar";
}

    And stylecow converts to:

    .foo:before {
    content: "bar";
}

  • fullscreen Enabled by default

    Fix Webkit/mozilla browsers that supports :-moz-full-screen and :-webkit-full-screen instead :-moz-fullscreen and :-webkit-fullscreen.

    You write:

    .foo:fullscreen {
    color: blue
}

    And stylecow converts to:

    .foo:-moz-full-screen {
    color: blue
}
.foo:-webkit-full-screen {
    color: blue
}
.foo:fullscreen {
    color: blue
}

  • vmin Enabled by default

    explorer < 10.0

    Add a ie9 fallback that support vm instead vmin, more info

    You write:

    .foo {
    width: 3vmin;
}

    And stylecow converts to:

    .foo {
    width: 3vm;
    width: 3vmin;
}

  • ffFillAvailable Enabled by default

    Fallback for firefox that supports the "-moz-available" property rather than "-moz-fill-available", more info

    You write:

    .foo {
    min-width: fill-available;
}

    And stylecow converts to:

    .foo {
    min-width: -moz-available;
    min-width: fill-available;
}

  • ieClip Enabled by default

    explorer < 8.0

    Fix the old syntax of clip for ie 6 and 7

    You write:

    .foo {
    clip: rect(5px, 40px, 45px, 5px);
}

    And stylecow converts to:

    .foo {
    clip: rect(5px, 40px, 45px, 5px);
    *clip: rect(5px 40px 45px 5px);
}

  • ieFloat Enabled by default

    explorer < 7.0

    Fix the double margin bug in ie6 on float block elements

    You write:

    .foo {
    float: left;
}

    And stylecow converts to:

    .foo {
    float: left;
    _display: inline;
}

Plugins to emulate css properties in old browsers

  • ieBackgroundAlpha Disabled by default

    explorer < 9.0

    Add support of rgba/hsla colors in backgrounds for explorer using ms filters more info. This plugin is disabled by default because it's not "safe", and the use of filters affects to performance. You can enable it with the parameter --enable=ieBackgroundAlpha

    You write:

    .foo {
    background: rgba(0, 0, 0, 0.5);
}

    And stylecow converts to:

    .foo {
    background: rgba(0, 0, 0, 0.5);
    filter: progid:DXImageTransform.Microsoft.gradient(startColorStr='#80000000', endColorStr='#80000000');
}

  • ieInlineBlock Enabled by default

    explorer < 8.0

    Add support for display:inline-block in explorer 6 and 7

    You write:

    .foo {
    display: inline-block;
}

    And stylecow converts to:

    .foo {
    display: inline-block;
    *zoom: 1;
    *display: inline;
}

  • ieLinearGradient Disabled by default

    explorer < 10.0

    Emulate the linear-gradient background in explorer using ms filters. This plugin is disabled by default because it's not "safe", and the use of filters affects to performance. You can enable it with the parameter --enable=ieBackgroundAlpha

    You write:

    .foo {
    background: linear-gradient(top, red, blue);
}

    And stylecow converts to:

    .foo {
    background: linear-gradient(top, red, blue);
    filter: progid:DXImageTransform.Microsoft.gradient(startColorStr='#FF0000', endColorStr='#0000FF');
}

  • ieMinHeight Enabled by default

    explorer < 7.0

    Add support for min-height property to ie 6, more info

    You write:

    .foo {
    min-height: 200px;
}

    And stylecow converts to:

    .foo {
    min-height: 200px;
    _height: 200px;
}

  • ieOpacity Enabled by default

    explorer < 9.0

    Add opacity support for explorer using filters

    You write:

    .foo {
    opacity: 0.5;
}

    And stylecow converts to:

    .foo {
    opacity: 0.5;
    filter: alpha(opacity=50);
}

  • ieTransform Disabled by default

    explorer < 9.0

    Add ms filters to emulate some 2d css transformations: rotate(), scale(), skew(), matrix(). This plugin is disabled by default because it's not "safe", and the use of filters affects to performance. You can enable it with the parameter --enable=ieBackgroundAlpha

    You write:

    .foo {
    transform: rotate(45deg);
}

    And stylecow converts to:

    .foo {
    transform: rotate(45deg);
    filter: progid:DXImageTransform.Microsoft.Matrix(sizingMethod="auto expand", M11 = 0.70710678118655, M12 = -0.70710678118655, M21 = 0.70710678118655, M22 = 0.70710678118655);
}

  • initial Enabled by default

    firefox < 19.0 chrome < 1.0 safari < 1.2 opera < 15.0

    Add support for the "initial" value

    You write:

    .foo {
    background-position: initial;
    height: initial;
}

    And stylecow converts to:

    .foo {
    background-position: 0 0;
    height: auto;
}

  • rem Enabled by default

    explorer < 9.0 firefox < 3.6 safari < 5.0 opera < 11.6 ios < 4.0

    Add a fallback in pixels for browsers not supporting rem values

    You write:

    .foo {
    font-size: 2rem;
}

    And stylecow converts to:

    .foo {
    font-size: 32px;
    font-size: 2rem;
}

    You can change the default value of 1rem (16px) defining a new font-size in :root or html. For example, you write:

    html {
    font-size: 1.2em;
}
.foo {
    font-size: 1rem;
}

    And stylecow converts to:

    html {
    font-size: 1.2em;
}
.foo {
    font-size: 19.2px;
    font-size: 1rem;
}
